Frequently Asked Questions

When you draw on the canvas or upload an image, the backend runs OpenCV contour counts. If it detects a single contour that is square or small, it routes it to the Keras MNIST digit or EMNIST character model. If it detects multiple spaced-out contours, it runs character segmentation. If the image has complex text layouts or lines, it calls the PyTorch-based EasyOCR page recognition module.

All prediction logs are written to local SQLite database schemas (`history.db`) automatically. The History page queries this database to let you audit and delete logged results.

In the **Prediction Detail** page, you can download recognition reports as a plain `.txt` file, a formatted `.pdf` document, or a Microsoft Word `.doc` file.

Quick Tips
  • Poor Accuracy: Draw characters of uniform height and space them slightly for clean OpenCV contours.
  • OCR Page Layouts: Ensure images are cropped closely to the writing with uniform lighting.
  • Word Document Downloads: Word files use standard HTML Word-MIME templates to run without client-side dependencies.
